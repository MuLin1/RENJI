# 医疗对话系统实验脚本说明书

## 一、系统概述

这四个脚本属于同一个**医疗对话决策树系统**的实验评测体系，围绕 `new_app.py` 核心模块展开。系统的核心能力有两个：

1. **树选择（Tree Selection）**：根据患者输入，从科室配置的多棵决策树中选出最匹配的一棵。
2. **条件节点判断（Condition Node Judgment）**：在决策树内部，判断某个条件节点是否满足。

四个脚本分别覆盖了 **数据标注 → 树选择评测 → 条件节点评测** 的实验流程。

---

## 二、各脚本详细说明

### 1. `annotation_script.py` —— 数据标注脚本（阶段一）

| 项目 | 说明 |
|------|------|
| **功能定位** | 批量标注/生成树选择的 ground truth |
| **依赖模块** | `paper_version/new_app.py` |
| **输入文件** | `reverted_requests.json`（患者请求数据） |
| **输出文件** | `annotation_results.json`（标注结果） |
| **使用模型** | 单一模型：`deepseek-v3.1`（阿里云 DashScope API） |

**工作流程：**
1. 加载患者请求数据
2. 初始化模型客户端，加载科室配置
3. 逐条调用 `select_best_matching_tree` 获取系统推荐的决策树
4. 记录每条样本的选择结果、处理时间、prompt 和模型原始输出等调试信息
5. 每 50 条保存中间结果，最终输出完整标注文件

**核心作用：** 用高质量大模型（deepseek-v3.1）为数据集打上"正确树"的标签，作为后续评测的 ground truth。

---

### 2. `tree_selection_test.py` —— 树选择测试脚本（阶段二·旧版） ⚠️

| 项目 | 说明 |
|------|------|
| **功能定位** | 评测树选择准确率（早期版本） |
| **依赖模块** | `new_frame/new_app.py`（⚠️ 旧版模块路径） |
| **输入文件** | `metadata.json`（测试数据，含 input 和 answer） |
| **输出文件** | `{MODEL_NAME}/tree_selection_results{N}.json` |
| **使用模型** | 单一模型：`deepseek-v3.1` |
| **科室** | 硬编码为 `心内科` |

**特征（与新版对比）：**
- 导入路径为 `new_frame`（旧版模块），新版已改为 `paper_version`
- 仅支持单模型串行测试
- 循环写的是 `range(1, 2)`（实际只跑 1 轮，注释写 5 轮，疑似调试遗留）
- 测试数据来源为手工准备的 `metadata.json`
- 无并行能力，无多模型支持

---

### 3. `evaluation_tree_test.py` —— 树选择评估脚本（阶段二·新版） ✅

| 项目 | 说明 |
|------|------|
| **功能定位** | 评测树选择准确率（成熟版本） |
| **依赖模块** | `paper_version/new_app.py`（新版模块路径） |
| **输入文件** | `pid_annotation_results.json`（标注脚本的输出） |
| **输出文件** | `evaluation_results_{model_name}.json`（每模型一个） |
| **使用模型** | 多模型批量对比（6 个模型） |

**测试的模型列表：**
- Llama-3.1-8B-Instruct
- qwen2.5-7b-instruct
- qwen2.5-3b-instruct
- qwen3-8b / qwen3-4b / qwen3-0.6b

**相比旧版的改进：**
- 支持**多模型批量评测**，一次运行对比所有模型
- 支持**多线程并行**评估（`ThreadPoolExecutor`，默认 5 并发）
- 对 Qwen3 系列模型封装了专用的 `Qwen3ClientWrapper` 类，自动注入 `enable_thinking: False`
- 使用标注脚本输出的 `pid_annotation_results.json` 作为 ground truth
- 线程安全的打印（`safe_print`）
- 科室信息从标注数据中动态读取，不再硬编码

---

### 4. `condition_node_judgment_test.py` —— 条件节点判断测试脚本（阶段三）

| 项目 | 说明 |
|------|------|
| **功能定位** | 评测决策树内部条件节点的判断准确率 |
| **依赖模块** | `paper_version/new_app.py` |
| **输入文件** | `label-to-id.json`（节点标注数据）+ `xnk_filtered_raw.jsonl`（对话测试数据） |
| **输出文件** | `evaluation_results_condition_node_{MODEL_NAME}.json` |
| **使用模型** | 单一模型：`Qwen3-0.6B`（本地部署，localhost:8080） |

**工作流程：**
1. 从 `label-to-id.json` 加载条件节点定义（每个节点有 content、label 列表、对应 PID 列表）
2. 从 `xnk_filtered_raw.jsonl` 加载真实对话数据
3. 对每个条件节点，用对应 PID 的对话数据调用 `judge_node_process`
4. 采用三档评分：**1.0（完全正确）/ 0.5（半对：应为"能"但判为"不能"）/ 0.0（错误）**
5. 每个节点重复 5 轮取平均准确率

**核心作用：** 评测小模型在决策树条件节点判断任务上的能力，这是树选择之后的下游子任务。

---

### 5. `xnk_pathway_dataset_builder.py` —— 心内科路径数据集构建脚本（数据准备）

| 项目 | 说明 |
|------|------|
| **功能定位** | 提取心内科急性病、多系统疾病及慢性高风险数据集 |
| **依赖模块** | 无（纯正则匹配） |
| **输入文件** | `xnk_filtered_raw.jsonl` |
| **输出文件** | `xnk_req2_..._dataset.jsonl` / `xnk_req4_..._dataset.jsonl` |

**数据筛选与映射方案：**

#### **(1) 需求2：急性病、多系统疾病**
主要针对发病急、危险程度高或由其他系统疾病引发的心血管症状。
- **急性高血压事件** ➡️ 映射树：`[有血压测试结果, 头晕or头胀or头痛]`
- **急性冠脉综合征** ➡️ 映射树：`[胸闷or胸痛or心脏不适or气促or呼吸困难, 有心电图检查]`
- **急性不稳定性心律失常** ➡️ 映射树：`[心悸or心跳不齐or心慌or昏厥, 心律失常诊断]`
- **多系统：甲状腺相关心律异常** ➡️ 映射树：`[心律失常诊断, 心悸or心跳不齐or心慌or昏厥]`
- **多系统：糖尿病心血管风险** ➡️ 映射树：`[胸闷or胸痛or心脏不适or气促or呼吸困难]`

#### **(2) 需求4：慢性病及高风险群体**
主要针对需要长期管理、复诊或具有叠加危险因素的群体。
- **慢性高血压长期管理** ➡️ 映射树：`[有高血压诊断史, 有血压测试结果]`
- **慢性血脂异常管理** ➡️ 映射树：`[有测血脂并且结果异常, 有血脂测试结果]`
- **ASCVD术后或确诊高风险** ➡️ 映射树：`[有冠脉CT检查, 胸闷or胸痛or心脏不适or气促or呼吸困难]`
- **多危险因素叠加高风险** ➡️ 映射树：`[有高血压诊断史, 有测血脂并且结果异常]`
- **心律失常高风险随访** ➡️ 映射树：`[心律失常诊断, 心悸or心跳不齐or心慌or昏厥]`

**使用方法：**
在终端直接运行：`python xnk_pathway_dataset_builder.py`

## 三、同阶段脚本对比：`tree_selection_test.py`（旧） vs `evaluation_tree_test.py`（新）

这两个脚本都是**树选择（Tree Selection）评测脚本**，属于同一阶段的新旧两个版本。

| 对比维度 | `tree_selection_test.py`（旧） | `evaluation_tree_test.py`（新） |
|----------|-------------------------------|-------------------------------|
| 模块路径 | `new_frame/new_app.py` | `paper_version/new_app.py` |
| 模型支持 | 单模型 | 6 个模型批量对比 |
| 并行能力 | 无（纯串行） | 多线程并行（5 workers） |
| 测试数据 | `metadata.json` | `pid_annotation_results.json` |
| Qwen3 适配 | 简单 wrapper 函数 | 完整 Wrapper 类体系 |
| 科室配置 | 硬编码 `心内科` | 从数据中动态获取 |
| 多轮测试 | 代码残留（写 5 实跑 1） | 正式支持 `NUM_ROUNDS` |
| 线程安全 | 无 | `safe_print` + `threading.local` |
| 结果输出 | 按轮次单文件 | 按模型单文件 |

**结论：** `evaluation_tree_test.py` 是 `tree_selection_test.py` 的全面升级版，适用于论文实验中的正式多模型对比评测。

## 四、整体实验流水线

```
阶段一：数据标注
  annotation_script.py
  用 deepseek-v3.1 为数据打标 → 输出 ground truth
        │
        ▼
阶段二：树选择评测
  tree_selection_test.py （旧版，已被替代）
  evaluation_tree_test.py（新版，正式使用）
  用多个候选模型做树选择，与 ground truth 对比 → 输出各模型准确率
        │
        ▼
阶段三：条件节点判断评测
  condition_node_judgment_test.py
  评测模型在决策树内部条件节点判断上的准确率 → 输出节点级评分
```

## 五、依赖与运行环境

- **Python 包**：`openai`, `json`, `threading`, `concurrent.futures`, `unittest.mock`
- **核心业务模块**：`new_app.py`（位于 `paper_version/` 目录）
- **API 服务**：阿里云 DashScope（线上模型）或本地 vLLM/Ollama 服务（本地模型）
- **数据文件**：
  - `reverted_requests.json` — 标注脚本输入
  - `metadata.json` — 旧版树选择测试输入
  - `pid_annotation_results.json` — 新版树选择测试输入（由标注脚本产出）
  - `label-to-id.json` + `xnk_filtered_raw.jsonl` — 条件节点测试输入
