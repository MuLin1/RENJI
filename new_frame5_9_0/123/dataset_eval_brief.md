评测阶段的核心输出是：给定前 `k` 轮对话，测试模型生成一句核心诊疗建议；再由评估模型给出语义一致性分数。示意如下：

```json

  "k": 2,

    "final_score_0_5": 4.5,
    "rationale": "预测覆盖了真值核心动作，方向一致。"
  }
}
```

## 2. 评估指标介绍

本项目的核心评估指标是**语义一致性分数**。

### 2.1 主指标

- `final_score_0_5`：评估模型给出的 0–5 分语义一致性分数
- `average_final_score_0_5`：数据集平均分
- `average_final_score_0_100`：把 0–5 分线性映射到 0–100 后的平均分，便于设定目标阈值

### 2.2 评分含义

- `4.0–5.0`：预测基本覆盖真值核心临床意图，且无明显冲突
- `3.5`：方向一致，有意义覆盖，但不够完整
- `3.0`：存在部分重合，但核心动作较弱
- `0.0–2.5`：核心意图缺失、场景偏离、明显冲突或存在风险

### 2.3 辅助统计指标

- `score_distribution`：0–5 分的分布
- `score_distribution_0_100`：0–100 分的分布
- `success_count / error_count / skipped_count`：评测成功、失败、跳过样本数

## 3. 测试模型与设置

这里的“测试模型”指：**在前 `k` 轮对话上生成诊疗结论的被测模型**。

### 3.1 测试模型

- 模型名：`Qwen/Qwen3.5-9B`
- 调用地址：`https://api.siliconflow.cn/v1`

### 3.2 测试设置

- 温度：`0`
- 请求超时：`60s`
- 并发数：`50`
- 上下文轮次上限：`6`
- 上下文字符上限：`5000`

### 3.3 测试任务定义

输入：

- 前 `k` 轮医患对话

输出：

- 一句简洁的核心诊疗建议 `prediction_text`

约束：

- 不输出推理过程
- 不补充对话中没有的信息
- 尽量保留检查、治疗、用药、复查、住院、手术、随访等核心动作

## 4. 评估模型

这里的“评估模型”指：**对 `prediction_text` 与 `ground_truth_text` 进行语义打分的模型**。

### 4.1 评分模型

- 模型名：`deepseek-v3.1`
- 调用地址：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- 用途：对预测结论与医生真值结论做非对称语义一致性评分

### 4.2 评分原则

- 重点看预测是否**覆盖并遵循**真值的核心意图
- 若预测更详细、更积极，但与真值不冲突，默认不轻易扣分
- 只在以下情况明显降分：
  - 核心动作缺失
  - 临床方向冲突
  - 场景明显偏离
  - 存在潜在安全风险

### 4.3 额外质量控制

在最终成集前，还使用同一模型 `deepseek-v3.1` 对**低分样本**做真值质量复核：

- 阈值：`final_score_0_5 <= 3.5`
- 目的：区分“模型预测差”与“真值本身低质量”
- 结果：剔除流程化、礼貌性、配药/登记/支付类、短问句类低价值真值样本
