## 对接“仁济药品库”RAG API 初版方案（集成到 `new_app.py`）

### Summary
- 在 `main_app_start_with_history()` 的“生成最终医生回复”前新增一个“药品库检索”步骤：先用 7B 模型判定是否需要查药品库并抽取 `query`，命中后调用 RAG 组 API `POST /search_drugs`，将结果以文本形式追加进 `report_info`（不改对外 API 返回结构）。

### Implementation Changes
- **在 `new_frame5_9_0/new_frame/new_app.py` 新增药品库对接模块（纯新增函数 + 最小改动接入点）**
  - 常量/配置
    - `DRUG_KB_URL`：默认 `http://172.20.137.65:13943/search_drugs`，允许用环境变量覆盖（例如 `RAG_DRUG_KB_URL`）。
    - 默认参数：`top_k=3`、`search_type="vector"`、`timeout=2~3s`。
  - LLM 触发与抽词（参考 `需求.md`）
    - 新增函数：`detect_drug_kb_need_and_query(patient_say, backend_history) -> (need: bool, query: str)`
    - 调用 `client_7b`（`enable_thinking=False`）让模型输出**严格 JSON**，例如：`{"need": true, "query": "..."}`。
    - 解析与兜底：
      - JSON 解析失败/字段缺失：降级为关键词规则（包含“用药/药物/副作用/禁忌/相互作用/剂量/说明书/孕妇/哺乳/儿童”等）决定 `need`，`query` 取 `patient_say` 的精简截断版。
      - `client_7b is None`：直接走关键词规则（避免因初始化失败导致整链路报错）。
  - 调用 RAG API
    - 新增函数：`search_drugs(query, top_k=3, search_type="vector") -> dict`
    - 实现用 Python 标准库 `urllib.request` 发起 `POST` JSON（避免引入 `requests` 新依赖）。
    - 对网络错误/非 200/JSON 解析失败：记录日志并返回空结果（不影响主流程）。
  - 结果压缩与注入主模型上下文
    - 新增函数：`format_drug_search_results(resp) -> str`
      - 仅保留 `drug_name` + 若干关键字段（如 `indications/contraindications/adverse_reactions/precautions/interactions`），每项做长度截断，避免 prompt 过长。
    - 在 `main_app_start_with_history()` 里、调用 `main_llm()` 之前：
      - 若 `need=True` 且检索有结果：把格式化文本追加到 `report_info`，用明确标签包裹，例如：
        - `<药品库检索结果 query="..."> ... </药品库检索结果>`
      - 全程只“增强上下文”，不改 `chat_api.py` 的返回字段（保持 API contract 不变）。
  - 日志
    - 记录：是否触发、抽取的 `query`、RAG 返回 `count`、是否降级/异常原因（不记录敏感信息；`query` 做长度截断）。

### Test Plan (手工验收为主)
- 启动服务后用 `/api` 验证 3 类场景：
  1. **应触发**：`patient_say="阿司匹林有什么副作用？能和华法林一起用吗？"` → 日志出现触发+RAG调用；模型回复中能结合药品库信息回答（至少能提及禁忌/相互作用/注意事项）。
  2. **不触发**：`patient_say="我胸口痛，怎么办"` → 日志显示不触发；不调用 RAG API。
  3. **RAG 不可用兜底**：临时把 `RAG_DRUG_KB_URL` 指到不可达地址 → 不应导致 500；仍能正常返回医生回复（只是无药品库补充）。
- 回归：原有摘要 pipeline、决策树推进、自由对话路径不受影响（重点看 `main_app_start_with_history()` 主流程是否仍能走通）。

### Assumptions / Defaults
- 默认对**所有科室**启用药品库触发（仅由 LLM/规则判定是否调用）。
- 不新增对外字段、不修改 `chat_api.py` 的响应结构；药品库信息只作为主模型上下文增强。
- 默认 `top_k=3`、`search_type="vector"`；后续如需可参数化再扩展。
