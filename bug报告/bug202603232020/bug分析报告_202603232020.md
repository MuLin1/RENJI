# MRI检查部位追问Bug分析报告 (2026-03-23)

## 1. 问题描述
全科决策树在处理患者明确表达“头部MRI”的情况下，错误返回了“MRI检查是哪个部位的？”，导致模型向患者发起冗余追问。

## 2. 根因分析
- **节点定义冲突**：
  - 触发文案来自 [MRI检查.json](file:///d:/研究生/仁济/new_frame5_9_0/new_frame/GP/trees/MRI检查.json) 的第2号节点。
  - 该节点的分支标签仅包含“关节”和“脊柱”，未覆盖“头部”，导致逻辑无法下钻。
- **树路由优先级问题**：
  - 系统中同时存在更精确的 [头部CT或MRI.json](file:///d:/研究生/仁济/new_frame5_9_0/new_frame/GP/trees/头部CT或MRI.json)。
  - 在 `select_best_matching_tree` 阶段，泛化的“MRI检查”树由于条件匹配过于宽泛，与专用树产生冲突，并可能被误选。
- **信息抽取反馈闭环缺失**：
  - 节点判断逻辑在面对非预设标签（如“头部”）时，直接退回“信息不足”状态，未能识别出已有信息与标签的语义鸿沟。

## 3. 排查路径
1. **日志回溯**：在 `logs/general.log` 中搜索 `best_tree_name`，确认本轮对话到底匹配到了哪棵树。
2. **候选树核对**：检查 `select_best_matching_tree` 逻辑输出，确认 `头部CT或MRI` 是否出现在候选列表中。
3. **策略校验**：核实 `judge_node_process` 采用的是哪种判断策略（1-4），以及 `evaluate_and_select_option` 是否因为标签缺失而返回 `None`。
4. **摘要一致性检查**：核实 `summarypipe` 生成的摘要是否保留了“头部/头颅”这一部位描述。

## 4. 建议方案 (不改动代码)
- **配置修正**：在 `GP_config.py` 中将 `MRI检查` 的触发条件细化，避免其拦截“头部MRI”的意图。
- **树定义完善**：在 `MRI检查.json` 的节点中增加“头部”或其他常见部位标签，并配置相应的跳转或决策节点。
- **Prompt微调**：优化 `batch_condition_judge_template`，增强模型对“部位实体”与“检查项”关联性的识别。
